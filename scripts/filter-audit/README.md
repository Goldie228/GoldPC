# Filter Audit — Аудит структуры фильтров каталога

Скрипт для генерации полного отчёта по всем доступным фильтрам категорий каталога.

## Назначение

Скрипт обходит все категории каталога через backend API и собирает структуру всех доступных фильтров (фасетов):
- Для **select-фильтров**: список всех возможных значений с форматированием
- Для **range-фильтров**: диапазоны min/max значений

Результат сохраняется в один markdown-файл для анализа командой.

## Использование

### Запуск

Из корня репозитория:

```bash
npm run filter-audit
```

Или напрямую:

```bash
npx tsx scripts/filter-audit/generate-filters-report.ts
```

### Требования

- Поднятый Catalog API (например, через `./scripts/dev-local.sh`)
- По умолчанию использует `http://localhost:5000`

### Переменные окружения

- `CATALOG_API_BASE` — базовый URL для Catalog API (по умолчанию `http://localhost:5000`)

Пример:

```bash
CATALOG_API_BASE=http://localhost:5000 npm run filter-audit
```

## Выход

Скрипт создаёт файл:

```
scripts/filter-audit/out/filters-report-YYYYMMDD-HHMMSS.md
```

### Структура отчёта

- **Заголовок** — дата генерации, API, общая статистика
- **Оглавление** — ссылки на все категории
- **Секции по категориям**:
  - Название категории, slug, количество товаров
  - Для каждого фильтра:
    - `select` — таблица со всеми значениями (сырые + форматированные)
    - `range` — диапазон min/max (с человекочитаемыми единицами где применимо)

```markdown
**Вариантов:** 45

|----------|-----------------|
| RTX 4090 | RTX 4090 |

```
