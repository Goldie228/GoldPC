# Уроки Code Review

## 📋 Назначение

Эта директория содержит документы с уроками, извлечёнными в процессе код-ревью проекта GoldPC. Каждый документ фиксирует проблему, найденную при ревью, её решение и влияние на проект.

## 🎯 Цели

- **Накопление знаний**: Сохранение опыта для будущих ревью
- **Предотвращение повторений**: Избежание одних и тех же ошибок
- **Обучение агентов**: Автоматическое внедрение уроков в контекст ИИ-агентов
- **Стандартизация**: Единый подход к документированию находок

## 📁 Структура

```
code-review/
├── README.md           # Этот файл
├── template.md         # Шаблон для новых записей
├── security/           # Проблемы безопасности
│   └── *.md
├── architecture/       # Архитектурные проблемы
│   └── *.md
├── performance/        # Проблемы производительности
│   └── *.md
└── best-practices/     # Нарушения лучших практик
    └── *.md
```

## 📝 Создание новой записи

1. Скопируйте `template.md` в соответствующую подкатегорию
2. Заполните все обязательные поля
3. Назовите файл по шаблону: `YYYY-MM-DD-краткое-название.md`
4. Создайте PR для добавления в базу знаний

## 🔍 Категории находок

| Категория | Описание | Примеры |
|-----------|----------|---------|
| **security** | Уязвимости, утечки данных, неправильная авторизация | SQL injection, hardcoded secrets, XSS |
| **architecture** | Нарушения Clean Architecture, циклические зависимости | Layer violations, God objects |
| **performance** | N+1 запросы, неоптимальные алгоритмы, memory leaks | Missing indexes, unbounded queries |
| **best-practices** | Нарушения coding standards, SOLID | Magic numbers, missing error handling |

## 🏷️ Метки серьёзности

| Метка | Описание | Действие |
|-------|----------|----------|
| 🔴 `critical` | Критическая уязвимость или блокер | Немедленное исправление |
| 🟠 `major` | Серьёзная проблема | Исправление в текущем спринте |
| 🟡 `minor` | Незначительная проблема | Исправление по возможности |
| 🔵 `suggestion` | Предложение по улучшению | Рекомендация |

## 🔄 Автоматическое внедрение

Уроки code-review автоматически внедряются в контекст агентов:

```yaml
# knowledge-base/knowledge-injection.yaml
agents:
  all:
    code_review_lessons:
      path: "lessons-learned/code-review/"
      auto_inject: true
      priority: high
```
