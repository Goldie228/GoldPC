## 📝 Регистрация

```mermaid
sequenceDiagram
    participant U as User
    participant FE as Frontend
    participant AS as AuthService
    participant PG as PostgreSQL
    participant SMTP as SMTP (Gmail)

    U->>FE: Заполнить форму регистрации
    FE->>FE: Client-side validation
    FE->>AS: POST /auth/register
    Note over AS: FluentValidation: email, password, name, phone
    
    AS->>PG: Check email uniqueness
    alt Email exists
        AS-->>FE: 409 Conflict (общая ошибка)
    else Email unique
        AS->>AS: Hash password (BCrypt, 12 rounds)
        AS->>PG: Create User
        AS->>PG: Create EmailVerificationToken
        AS->>PG: Create RefreshToken
        AS->>SMTP: Send verification email
        AS-->>FE: 201 Created
        Note over AS,FE: { accessToken, refreshToken, user }
        
        FE->>FE: Store tokens in authStore
        FE-->>U: Показать dashboard
        
        U->>FE: Open email, click link
        FE->>AS: GET /auth/verify-email?token=...
        AS->>PG: Find + verify token
        AS->>PG: Set IsEmailVerified=true
        AS-->>FE: Email verified page
    end
```

### Валидация (FluentValidation)

```csharp
public class RegisterValidator : AbstractValidator<RegisterRequest>
{
    public RegisterValidator()
    {
        RuleFor(x => x.Email).EmailAddress().MaximumLength(256);
        RuleFor(x => x.Password)
            .MinimumLength(8)
            .Matches("[A-Z]").WithMessage("Требуется заглавная буква")
            .Matches("[a-z]").WithMessage("Требуется строчная буква")
            .Matches("[0-9]").WithMessage("Требуется цифра")
            .Matches("[^a-zA-Z0-9]").WithMessage("Требуется спецсимвол");
        RuleFor(x => x.FirstName).NotEmpty().MaximumLength(100);
        RuleFor(x => x.LastName).NotEmpty().MaximumLength(100);
        RuleFor(x => x.Phone).Matches(@"^\+375\d{9}$");
    }
}
```

## 🔑 Логин

```mermaid
sequenceDiagram
    participant U as User
    participant FE as Frontend
    participant AS as AuthService
    participant PG as PostgreSQL

    U->>FE: Ввести email + password
    FE->>AS: POST /auth/login
    
    AS->>PG: Find User by email
    
    alt User not found
        AS-->>FE: 401 "Неверный email или пароль"
    else User found
        AS->>AS: Check LockedUntil
        alt Account locked
            AS-->>FE: 423 "Аккаунт заблокирован до {time}"
        else Not locked
            AS->>AS: Verify BCrypt password
            
            alt Password invalid
                AS->>PG: Increment FailedLoginAttempts
                AS->>PG: Record LoginHistory (failed)
                
                alt 5 failures reached
                    AS->>PG: Set LockedUntil = UTC+15min
                    AS-->>FE: 423 "Аккаунт заблокирован на 15 мин"
                else <5 failures
                    AS-->>FE: 401 "Неверный email или пароль"
                end
                
            else Password valid
                AS->>PG: Reset FailedLoginAttempts = 0
                AS->>PG: Create RefreshToken
                AS->>PG: Record LoginHistory (success)
                AS-->>FE: 200 OK
                Note over AS,FE: { accessToken, refreshToken, user }
                
                FE->>FE: Store in authStore (Zustand)
                FE->>U: Redirect to dashboard
            end
        end
    end
```

## 🔄 Refresh Token Flow

```mermaid
sequenceDiagram
    participant FE as Frontend
    participant AS as AuthService
    participant PG as PostgreSQL

    Note over FE,PG: Access Token expires after 15 min

    FE->>FE: Axios interceptor detects 401
    FE->>AS: POST /auth/refresh { refreshToken }
    
    AS->>PG: Find RefreshToken by value
    AS->>AS: Verify: not revoked, not expired
    
    alt Valid token
        AS->>PG: Revoke old token (RevokedAt = now)
        AS->>PG: Create new RefreshToken
        AS-->>FE: { accessToken, newRefreshToken }
        
        FE->>FE: Update tokens in store
        FE->>FE: Retry original request
    else Token revoked (possible theft!)
        AS->>PG: Revoke ALL user tokens
        AS-->>FE: 401 "Session expired"
        FE->>FE: Clear auth state, redirect to login
    else Token expired
        AS-->>FE: 401 "Token expired"
        FE->>FE: Redirect to login
    end
```

## 📱 Аутентификация на фронтенде

### Zustand Store

```typescript
interface AuthState {
  user: User | null;
  accessToken: string | null;
  refreshToken: string | null;
  isAuthenticated: boolean;
  
  login: (email: string, password: string) => Promise<void>;
  register: (data: RegisterData) => Promise<void>;
  logout: () => Promise<void>;
  refresh: () => Promise<void>;
}

// Хранение: Zustand store (не localStorage)
// Токены хранятся в памяти, не в localStorage/cookies
```

### Axios Interceptors

```typescript
// Request: добавляем Bearer token
api.interceptors.request.use(config => {
  const token = useAuthStore.getState().accessToken;
  if (token) config.headers.Authorization = `Bearer ${token}`;
  return config;
});

// Response: автоматический refresh при 401
api.interceptors.response.use(
  res => res,
  async error => {
    if (error.response?.status === 401 && !error.config._retry) {
      error.config._retry = true;
      await useAuthStore.getState().refresh();
      return api(error.config);
    }
    return Promise.reject(error);
  }
);
```
