## 📋 Prerequisites

| Компонент | Версия | Проверить |
|-----------|--------|-----------|
| **Docker** | 20.10+ | `docker --version` |
| **Docker Compose** | 2.0+ | `docker compose version` |
| **Node.js** | 20+ | `node --version` |
| **.NET SDK** | 8.0 | `dotnet --version` |
| **Git** | любая | `git --version` |

---

## ⚡ Быстрый старт

```bash
# 1. Клонировать репозиторий
git clone https://github.com/goldie1k/GoldPC.git
cd GoldPC

# 2. Настроить переменные окружения
cp .env.example .env
# Отредактировать .env при необходимости

# 3. Запустить инфраструктуру (БД, Redis, RabbitMQ)
make infra
# или: docker compose -f docker/docker-compose.yml up -d

# 4. Запустить всё (frontend + backend)
make dev
# или: npm run dev:local
```

## 🐳 Docker Compose инфраструктура

```yaml
# docker/docker-compose.yml — Development окружение
services:
  postgres:      # :5434
  postgres-replica:  # :5435
  redis:         # :6379
  rabbitmq:      # :5672 (UI :15672)
  adminer:       # :9090
  prometheus:    # :9090
  grafana:       # :3002
  jaeger:        # :4317 (gRPC OTLP)
```

## 🛠️ Makefile команды

```bash
make infra       # Запуск инфраструктуры (Docker)
make dev         # Полный dev-запуск (всё сразу)
make build       # Сборка всех компонентов
make test        # Все тесты
make db-seed     # Заполнение БД тестовыми данными
make clean       # Остановка и очистка
make logs        # Просмотр логов
```

## 🐛 Возможные проблемы

### Docker не стартует
```bash
# Проверить, что Docker daemon запущен:
sudo systemctl status docker
# Или перезапустить:
sudo systemctl restart docker
```

### Порты заняты
```bash
# Проверить занятые порты:
sudo lsof -i :5434
# Изменить порты в .env при конфликте
```

### NPM install ошибки
```bash
# Очистить кэш и переустановить:
rm -rf node_modules
npm cache clean --force
npm install
```

### Миграции БД не применились
```bash
# Применить миграции вручную:
make db-migrate
# или:
cd src/backend/AuthService
dotnet ef database update
```
