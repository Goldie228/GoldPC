## 🏗️ Architecture (Архитектура и дизайн)

### Структура и зависимости

- [ ] Изменения соответствуют общей архитектуре системы
- [ ] Нет нарушения принципов SOLID
  - [ ] **S**ingle Responsibility — каждый класс имеет одну ответственность
  - [ ] **O**pen/Closed — открыт для расширения, закрыт для модификации
  - [ ] **L**iskov Substitution — подтипы могут заменять базовые типы
  - [ ] **I**nterface Segregation — интерфейсы специфичны и не слишком большие
  - [ ] **D**ependency Inversion — зависимость от абстракций, не от конкретных реализаций
- [ ] Зависимости направлены правильно (Clean Architecture)
  - [ ] Domain не зависит от Infrastructure
  - [ ] Application не зависит от Infrastructure
  - [ ] Presentation зависит только от Application
- [ ] Паттерны использованы уместно и оправданно
- [ ] Нет циклических зависимостей

### Модульность

- [ ] Новый код размещён в правильном модуле/слое
- [ ] Публичные API чётко определены
- [ ] Инкапсуляция соблюдена (internal/private модификаторы)
- [ ] Нет неуместных связей между модулями

### Масштабируемость

- [ ] Решение масштабируется при росте нагрузки
- [ ] Нет узких мест производительности в архитектуре
- [ ] Stateless дизайн там, где это применимо

## 🔒 Security (Безопасность)

### OWASP Top 10

- [ ] **A01:2021 — Broken Access Control**
  - [ ] Авторизация проверяется для всех защищённых эндпоинтов
  - [ ] Principle of Least Privilege соблюдён
  - [ ] Нет прямых ссылок на объекты без проверки прав

- [ ] **A02:2021 — Cryptographic Failures**
  - [ ] Чувствительные данные шифруются в покое и при передаче
  - [ ] Используются современные алгоритмы шифрования
  - [ ] Нет hardcoded секретов в коде

- [ ] **A03:2021 — Injection**
  - [ ] Нет SQL injection (используются параметризованные запросы)
  - [ ] Нет XSS (данные экранируются)
  - [ ] Нет Command Injection
  - [ ] Нет LDAP/XML injection

- [ ] **A04:2021 — Insecure Design**
  - [ ] Модель угроз рассмотрена
  - [ ] Безопасность встроена в дизайн

- [ ] **A05:2021 — Security Misconfiguration**
  - [ ] Дефолтные учётные данные изменены
  - [ ] Debug режим отключён в production
  - [ ] Error messages не раскрывают чувствительную информацию

- [ ] **A06:2021 — Vulnerable Components**
  - [ ] Зависимости проверены на известные CVE
  - [ ] Используются актуальные версии библиотек

- [ ] **A07:2021 — Identification and Authentication Failures**
  - [ ] Пароли хранятся в хэшированном виде (bcrypt, Argon2)
  - [ ] Сессии управляются безопасно
  - [ ] Multi-factor authentication там, где применимо

- [ ] **A08:2021 — Software and Data Integrity Failures**
  - [ ] CI/CD pipeline защищён
  - [ ] Автоматические обновления верифицированы

- [ ] **A09:2021 — Security Logging and Monitoring Failures**
  - [ ] Аудит-логи для критических операций
  - [ ] Логи не содержат чувствительных данных
  - [ ] Alerting настроен на аномалии

- [ ] **A10:2021 — Server-Side Request Forgery (SSRF)**
  - [ ] Внешние URL валидируются
  - [ ] Whitelist разрешённых доменов

### Валидация данных

- [ ] Входные данные валидируются на сервере
- [ ] Валидация на уровне типов (DTO, Value Objects)
- [ ] Boundary validation (на границах слоёв)
- [ ] Sanitization пользовательского ввода

### Аутентификация и авторизация

- [ ] JWT токены валидируются корректно
- [ ] Token refresh реализован безопасно
- [ ] Logout инвалидирует токены
- [ ] Role-based access control работает правильно

## 💼 Business Logic (Бизнес-логика)

### Корректность алгоритмов

- [ ] Алгоритм реализован в соответствии с требованиями
- [ ] Формулы и вычисления проверены
- [ ] Edge cases обработаны
- [ ] Граничные условия учтены (off-by-one, пустые коллекции, null)

### Обработка ошибок

- [ ] Ошибки обрабатываются gracefully
- [ ] Пользователь получает понятные сообщения об ошибках
- [ ] Ошибки логируются с достаточным контекстом
- [ ] Нет swallowed exceptions (пустых catch блоков)
- [ ] Correct error propagation между слоями

### Транзакционность

- [ ] Транзакции используются где необходимо
- [ ] Deadlocks исключены
- [ ] Idempotency обеспечена где применимо
- [ ] Race conditions обработаны

### Производительность бизнес-логики

- [ ] N+1 queries отсутствуют
- [ ] Запросы оптимизированы
- [ ] Кэширование используется где уместно
- [ ] Async/await используется правильно

## 🎨 UX/UI (Пользовательский интерфейс)

### Дизайн и визуал

- [ ] Интерфейс интуитивно понятен
- [ ] Дизайн соответствует дизайн-системе/guidelines
- [ ] Консистентность с остальным приложением
- [ ] Responsive design (адаптивность)
- [ ] Визуальная иерархия соблюдена

### Accessibility (Доступность)

- [ ] WCAG 2.1 Level AA compliance
- [ ] Keyboard navigation поддерживается
- [ ] Screen reader friendly (ARIA атрибуты)
- [ ] Цветовой контраст достаточен (4.5:1 для текста)
- [ ] Не используется только цвет для индикации

### Usability (Удобство использования)

- [ ] Обратная связь на действия пользователя (loading states, success/error)
- [ ] Error messages понятны и помогают исправить проблему
- [ ] Confirmation dialogs для деструктивных действий
- [ ] Undo/Redo где применимо
- [ ] Progressive disclosure (постепенное раскрытие сложности)

### Формы и ввод данных

- [ ] Валидация на клиенте с понятными сообщениями
- [ ] Field labels понятны
- [ ] Required fields отмечены
- [ ] Autocomplete работает корректно
- [ ] Focus management правильный

---

## 📖 Maintainability (Поддерживаемость)

### Читаемость кода

- [ ] Код понятен без избыточных комментариев
- [ ] Имена переменных, функций, классов говорящие
- [ ] Функции имеют разумную длину (< 30 строк)
- [ ] Файлы имеют разумный размер (< 500 строк)
- [ ] Нет "магических чисел" без объяснения

### Документация

- [ ] XML doc comments для публичных API (C#)
- [ ] JSDoc/TSDoc для публичных функций (TypeScript)
- [ ] README для новых модулей
- [ ] ADR для архитектурных решений

### Тестируемость

- [ ] Код спроектирован для тестирования
- [ ] Dependency injection используется
- [ ] Mockable interfaces
- [ ] Deterministic tests (нет flaky тестов)

---

## 🧪 Testing (Тестирование)

### Покрытие тестами

- [ ] Unit тесты для новой логики
- [ ] Integration тесты для взаимодействия компонентов
- [ ] E2E тесты для критических сценариев
- [ ] Coverage ≥ 70%

### Качество тестов

- [ ] Тесты независимы друг от друга
- [ ] Тесты детерминированы
- [ ] Arrange-Act-Assert pattern
- [ ] Тесты понятны (говорящие названия)
- [ ] Edge cases протестированы
