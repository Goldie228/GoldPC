# Отчёт: множественный выбор фильтров в конструкторе ПК

**Дата:** 2026-04-04  
**Окружение:** `http://localhost:5173/pc-builder`, API через Vite proxy  
**Код:** [`FilterSidebar.tsx`](src/frontend/src/components/catalog/FilterSidebar.tsx) (`filterType === 'select'` — чекбоксы, массив в `selectedSpecifications`)

Формат: `[ОК]` / `[БАГ]` по запросу.

## Сводка

| Сценарий | Вердикт | Комментарий |
|----------|---------|-------------|
| 1 Бренды (AMD+Intel, чекбоксы не исчезают) | [ОК] без выбранного CPU с сокетом AM4/AM5 | В пикере CPU при пустой сборке `restrictedManufacturerPlatform` не передаётся — в UI видны как минимум **AMD** и **Intel** (снимок модалки). После выбора CPU с **AM4/AM5** для слотов CPU/MB включается фильтр по подстроке в имени — см. [БАГ] ниже (не про мультивыбор, а про полноту списка). |
| 2 Чипсет, мультивыбор без CPU | [ОК] по коду | При отсутствии `restrictedSpecValues.chipset` список опций не срезается — можно отметить несколько чипсетов (OR через массив). |
| 3 Сокет МП при выбранном CPU | [ОК] по коду | Если в `selectedSpecifications` сокет пришёл строкой из контекста (`effectiveSpecs`), срабатывает ветка `isLocked` — в UI остаётся **одна** опция, остальные **скрыты** (стр. 686–689 `FilterSidebar.tsx`). |
| 4 Форм-фактор корпуса при micro-ATX МП | [ОК] (ранее + код) | `caseFormFactorsForMB` + `effectiveSpecs.formFactor`; несовместимые FF не попадают в выдачу. |
| 5 Тип памяти RAM при МП DDR4 | [ОК] по коду | Для `type` с непустым `allowed` применяется `isMatchingDDRType` — узкий набор DIMM; мультивыбор нескольких несовместимых типов недоступен. |
| 6 Тип памяти RAM без CPU/MБ | [ОК] по коду | Без жёсткого `locked`/`allowed` — мультивыбор через чекбоксы. |
| 7 Охлаждение: типы кулер/СЖО | [ОК] по коду | `typeFilter` задаёт массив в `effectiveSpecs`; фильтрация по `allowed` оставляет только перечисленные типы; внутри набора — мультивыбор. |
| 8 Цена | [ОК] по коду | Диапазон `range`, поля мин/макс и слайдер. Полный интерактив в браузере не записывался скриншотом. |

## [ОК] Сценарий 1 (часть): мультивыбор брендов AMD + Intel при пустой сборке

- **Ожидание:** Два бренда можно отметить; оба остаются в списке; товары сужаются по ИЛИ.  
- **Реальность:** При пустом конструкторе в пикере CPU `mfrPlatform` не задан, список брендов не режется по подстроке; в accessibility-деревне видны пункты **AMD** и **Intel** одновременно с остальными фасетами. Логика чекбоксов — добавление в массив (`onChange` в `FilterSidebar`).

---

## [БАГ] Сценарии 1, 4, 8: не «все бренды категории», а урезание по платформе (AM4/AM5 / Intel LGA)

- **Сценарий:** 1 (после выбора AMD AM4 CPU), 4, 8  
- **Ожидание:** В модалке материнской платы (и при необходимости CPU) в блоке «Бренды» доступны производители материнок: ASUS, Gigabyte, MSI, ASRock и т.д.; при выборе одного бренда остальные **остаются в списке чекбоксов**.  
- **Реальность:** [`PCBuilderPage.tsx`](src/frontend/src/pages/PCBuilderPage/PCBuilderPage.tsx) передаёт `restrictedManufacturerPlatform` для `cpu` и `motherboard` при вычисленном `mfrPlatform`; [`FilterSidebar.tsx`](src/frontend/src/components/catalog/FilterSidebar.tsx) оставляет только производителей с подстрокой `amd` или `intel` в **названии**. Мультивыбор из **оставшихся** двух имён возможен, но это не соответствует таблице ожиданий («Всегда все бренды категории»).

---

## [ОК] Сценарий 2: чипсеты B550 + A520 без выбранного CPU

- **Ожидание:** Мультивыбор, OR по товарам, прочие чипсеты на экране.  
- **Реальность:** Без контекста CPU `restrictedSpecValues.chipset` для материнки обычно не задан — список не фильтруется до подмножества; несколько значений в `selectedSpecifications['chipset']` поддерживаются.

---

## [ОК] Сценарий 3: сокет материнки при выбранном Ryzen AM4 — один вариант

- **Ожидание:** Только AM4, нельзя выбрать другой сокет.  
- **Реальность:** Жёсткая привязка через строку в спеках → ветка `isLocked` — показывается только совпадающая опция.

---

## Примечание по «исчезновению» опций

В [`FilterSidebar.tsx`](src/frontend/src/components/catalog/FilterSidebar.tsx) при **непустом** `restrictedSpecValues[key]` список опций **фильтруется** (`finalOptions = cleanOptions.filter(...)`) — опции вне `allowed` **не отображаются**. Это ожидаемо для сокета/чипсета при выбранном CPU (сценарий 3 и контекстные ограничения), но **отличается** от сценария 2 без CPU, где `allowed` для чипсета не задан.

---

## Сценарии 5–8

- **5:** При выбранной МП с DDR4 контекст ограничивает тип памяти — один корректный тип DIMM; **соответствует** ожиданию «только один вариант».  
- **6:** Без МП/CPU мультивыбор типов в RAM поддерживается реализацией `select`.  
- **7:** Типы охлаждения задаются `typeFilter` в [`PCBuilderPage.tsx`](src/frontend/src/pages/PCBuilderPage/PCBuilderPage.tsx); лишние типы (вентилятор корпуса, паста) должны отсутствовать в фасетах после ограничения — зависит от строк в каталоге.  
- **8:** Цена — `filterType === 'range'`; **OK** по архитектуре.

---
