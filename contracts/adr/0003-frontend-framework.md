# ADR-0003: Выбор React/Vite для Frontend

## Статус
Принято

## Контекст
Для проекта GoldPC необходимо выбрать фреймворк для разработки клиентской части веб-приложения.

### Требования
- SPA (Single Page Application) для богатого UI
- Быстрая разработка и hot-reload
- Интеграция с REST API
- Типобезопасность (TypeScript)
- Компонентный подход для переиспользования

### Ограничения
- Команда знакома с React
- Требуется быстрая time-to-market
- Интеграция с ASP.NET Core backend

### Текущее состояние проекта
Проект уже использует React с Vite:
- **React 19.2.4** — последняя версия библиотеки
- **Vite 8.0.0** — современный сборщик
- **TypeScript 5.9.3** — типобезопасность
- **React Router 7.13.1** — маршрутизация
- **Axios 1.13.6** — HTTP клиент

```json
{
  "dependencies": {
    "react": "^19.2.4",
    "react-dom": "^19.2.4",
    "react-router-dom": "^7.13.1",
    "axios": "^1.13.6"

  "devDependencies": {

  }

```

## Решение
Выбран **React 19 + Vite 8 + TypeScript** как стек для frontend разработки.

### Обоснование

#### React
1. **Компонентная модель** — переиспользование UI элементов
2. **React Hooks** — управление состоянием без классов
3. **Экосистема** — богатый выбор библиотек (MUI, Redux, React Query)
4. **Рынок труда** — большое количество разработчиков
5. **React 19** — новые фичи (Server Components готовы к будущему)

#### Vite
1. **ESBuild** — молниеносная сборка (в 10-100 раз быстрее Webpack)
2. **Hot Module Replacement** — мгновенные обновления при разработке
3. **Native ES modules** — современный подход к модулям
4. **Простая конфигурация** — минимум настроек

#### TypeScript
1. **Типобезопасность** — ошибки на этапе компиляции
2. **IntelliSense** — автодополнение в IDE
3. **Рефакторинг** — безопасное переименование

## Последствия

### Положительные
- ✅ Быстрая разработка с Vite HMR
- ✅ Типобезопасность с TypeScript
- ✅ Компонентный подход для UI
- ✅ Богатая экосистема React
- ✅ Отличная интеграция с REST API через Axios
- ✅ Dependency Cruiser для проверки архитектуры

### Отрицательные
- ⚠️ Кривая обучения TypeScript для новых разработчиков
- ⚠️ React 19 — новая версия, возможны изменения

### Риски
- **Низкий риск** — React стандарт индустрии
- Митигация: использование стабильных паттернов и библиотек

## Альтернативы

| Альтернатива | Плюсы | Минусы | Причина отклонения |
|--------------|-------|--------|---------------------|
| **Vue 3** | Простой learning curve, отличный DX | Меньше рынок труда, меньшая экосистема | Команда не имеет опыта |
| **Angular** | Полный framework, TypeScript-first | Сложность, boilerplate, кривая обучения | Избыточен для проекта |
| **Svelte** | Компиляция, малый bundle | Меньшая экосистема, меньше разработчиков | Риск для production |
