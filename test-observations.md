
Про# PC Builder — Test Observations
Date: 2026-04-06

## Issues Found

### BUG-1: Фильтры не фильтруют товары (КРИТИЧЕСКИЙ)
**Где**: Модал выбора Процессор (и возможно все остальные слоты)
**Что**: При переключении любых фильтров (чекбоксы, range-слайдеры, радио-кнопки, текстовый поиск) число результатов всегда показывает "Найдено: 12 из 164" — не меняется.
**Как воспроизвести**:
1. Открыть модал выбора Процессор
2. Выбрать любой фильтр (например, Бренд = AMD в чекбоксах)
3. Чекбокс визуально переключается, но "Найдено: 12 из 164" остаётся без изменений
**Возможная причина**: События onChange/onClick не доходят до React-стейта. DOM-манипуляции с чекбоксами не триггерят React. Возможно проблема в обработчике событий фильтра.
**Проверено**:
- Чекбоксы (Сокет, Бренды, Наличие) — не фильтруют ✗
- Range sliders (Цена, Дата выхода, Ядра, TDP и т.д.) — не фильтруют ✗
- Текстовый поиск "intel core i5" — не фильтрует ✗
- Rating (radio group) — не фильтрует ✗
- "Сбросить фильтры" — частично работает: "В наличии" остаётся активным после сброса

### BUG-2: Кнопка "Сбросить фильтры" не сбрасывает "В наличии"
**Где**: Модал выбора Процессор
**Что**: После нажатия "Сбросить фильтры" чекбокс "В наличии" остаётся отмеченным, и "Найдено: 12 из 164" показывает только товары в наличии.

### BUG-3: Price filter показывает 0 BYN — 0 BYN
**Где**: Модал, фильтр "Цена"
**Что**: Range слайдер и значения min/max показывают 0 BYN — 0 BYN. Должны показывать реальный ценовой диапазон товаров (например, мин. и макс. цену среди 164 процессоров).
**Серъёзность**: Высокая — делает фильтр цены бесполезным.

### BUG-4: Обычные клики Playwright таймаутятся на кнопках "Выбрать"
**Где**: Главная страница слотов комплектующих
**Что**: `locator.click()` на кнопке "Выбрать" таймаутится (element не stable), работает только `evaluate(el => el.click())`.
**Что это значит для реальных пользователей**: Может быть проблема с CSS-анимацией или постоянным ре-рендером, мешающим клику. В обычном браузере кнопки могут работать, но это указывает на нестабильность UI.

### BUG-5: Кнопка "Выбрать" в карточке не обновляет предпросмотр
**Где**: Модал выбора Процессор
**Что**: Клик по кнопке "Выбрать" в карточке товара (AMD Athlon X4 970, 41.48 BYN) не обновляет панель предпросмотра — текст остаётся "Выберите товар из списка, здесь появятся его характеристики."
**Детали**: Кнопка `_cardTitleBtn_vmmnq_264` имеет textContent = название товара ("Процессор AMD Athlon X4 970"), type="button", disabled=false, но клик не вызывает предпросмотр.

### BUG-6: Карточки товаров недоступны для Playwright click()
**Где**: Модал выбора любого компонента
**Что**: `locator.click()` таймаутится на кнопках карточек. Работает только через `evaluate(el => el.click())`. Возможно CSS-анимация или проблема с hit-testing.

### BUG-7: Карточка товара не выделяется визуально после выбора
**Где**: Модал выбора Процессор
**Что**: После клика "Выбрать" на карточке preview обновляется, но сама карточка не получает visual-статус "selected"/"active". Пользователь не видит, какой товар сейчас выбран.

### Примечание: Кнопка "Выбрать" в превью работает
Клик на кнопку "Выбрать" в панели предпросмотра закрывает модал и обновляет слот на главной. Слот получает класс "component-slot--selected", показывает название товара, характеристики, кнопки "Снять" и "Изменить".

## Тесты здравого смысла (2026-04-06)

### BUG-12: КРИТИЧЕСКИЙ — Можно добавить 8 планок RAM на материнскую плату с 2 слотами
**Где**: Главная страница PC Builder, слоты ОЗУ (1-8)
**Что**: Приложение позволяет установить 8 планок оперативной памяти на материнскую плату BIOSTAR A520MHP Ver. 6.0, которая имеет **только 2 слота** (`memory_slots: 2` по спецификации).
**Детали**:
- Мат. плата поддерживает DDR4, до 64GB, максимум 2 модуля
- В сборке было 8 одинаковых планок AMD Radeon R7 Performance 4GB DDR4 (158,44 BYN каждая)
- Итого 8x4 = 32GB RAM, но физически плата НЕ поддерживает больше 2 модулей
- Это делает сборку физически невозможной
**Что должно быть**: Приложение должно проверять `memory_slots` материнской платы и ограничивать количество слотов RAM. Если у платы 2 слота — максимум 2 модуля RAM. Предупреждение о несовместимости.

### BUG-13: Нет обратной связи от "Сохранить" и "Печать"
**Где**: Боковая панель, кнопки "Сохранить" / "Печать"
**Что**:
- "Сохранить" — клик не показывает никакого уведомления. Нет подтверждения что сборка сохранена. (ранее BUG-8)
- "Печать" — клик не вызывает диалог печати браузера и не показывает уведомления. Пользователь не понимает, что произошло. (ранее BUG-9)
**Возможная причина**: Кнопки могут вызывать `window.print()` или сохранять в `localStorage` без визуальной обратной связи.

### BUG-14: Производительность — 0 FPS в играх (без GPU)
**Где**: Боковая панель, секция "Производительность"
**Что**: Игровой "Итого: 18". 1080p: 0 FPS, 1440p: 0, 4K: 0. Рабочий "Итого: 16".
**Анализ**:
- Без видеокарты 0 FPS в играх — логично, особенно с CPU без iGPU (Athlon X4 970)
- Система корректно предупреждает: "Процессор не имеет встроенной графики"
- Рабочая оценка 16 — корректна для базовой конфигурации без GPU
**Вывод**: Это корректное поведение, но пользователь может быть запутан. 0 FPS без объяснения (кроме мелкого предупреждения выше) выглядит как баг.

### BUG-15: UI дублирования — tooltip описания отображаются постоянно, а не по hover
**Где**: Все слоты комплектующих на главной странице
**Что**: Каждый слот имеет `<div class="component-slot__tooltip">` с описанием типа (напр. "Быстрая память для временных данных. Объём и тайминги влияют..."). Эти элементы **перманентно видны** на каждом слоте — не скрываются до hover.
**Проблема**: Это не hover-tooltips. Каждый слот показывает свою подсказку в виде маленького текстового блока над контентом. Это захламляет интерфейс и дублирует информацию.

### BUG-16: Несоответствие количества слотов RAM в UI
**Где**: Главная страница PC Builder
**Что**:
- Приложение генерирует 8 слотов RAM (ОЗУ (1) ... ОЗУ (8))
- Материнская плата BIOSTAR A520MHP имеет **только 2 слота** DDR4
- UI позволяет пользователю заполнить все 8 слотов RAM
- В sidebar суммарно указано 8 слотов ОЗУ, каждый по 158,44 BYN
**Ожидаемое поведение**: Количество слотов RAM должно ограничиваться спецификацией материнской платы. Если выбранная мат. плата имеет 2 слота RAM, максимум 2 слота должно быть в UI.

### BUG-17: Кнопка "В корзину" добавляет товары без визуальной обратной связи
**Где**: Корзина (sidebar), клик "В корзину"
**Что**: После клика "В корзину", корзина содержит **32 товара** (изначально было 0). Нет никакого toast/уведомления о том, что товары добавлены. Пользователь не видит подтверждения успешного действия.
**Детали**:
- CPU (1) + Мат.плата (1) + RAM (8) + SSD (1) + БП (1) + Кулер (1) + Монитор (1) + Мышь (1) = 15 SKU
- 32 товара в корзине — вероятно включая предыдущие тестовые сессии
**Ожидаемое поведение**: Показать toast "15 товаров добавлено в корзину".

### BUG-18: Кнопки +/− для RAM — неочевидное поведение дублирования
**Где**: Главная страница PC Builder, слот RAM (1)
**Что**: Кнопки +/− рядом с количеством RAM (показывает "−8+") позволяют изменить количество одинаковых модулей в слоте.
**Проблема**:
1. Кнопка + на первом RAM слоте отключена (disabled) при qty=8
2. При нажатии − количество уменьшается, + активируется
3. UI не объясняет, что + создаст дубликат того же товара
4. Неочевидно зачем пользователю дублировать один и тот же модуль
5. Нет ограничения по количеству слотов мат. платы
**Ожидаемое поведение**: Убрать кнопки +/− или показать понятное объяснение. Лучше — ограничить количество слотов RAM спецификацией мат. платы.

### BUG-19: Охлаждение не имеет спецификаций для валидации
**Где**: Кулер ID-Cooling DK-01S в сборке
**Что**: `specifications: {}` (пустой объект в localStorage) — нет данных о сокете, рассеиваемой мощности, хотя в описании указано: "Сокет: LGA1700, AM4... Рассеиваемая мощность: 65 Вт".
**Проблема**: Невозможно автоматически проверить совместимость кулера с CPU, потому что `specifications` пустой. Приложение не может валидировать поддержку сокета AM4 или достаточно ли 65W TDP кулера.

### BUG-20: Блок питания 350W — минимальный, нет предупреждения об апгрейде
**Где**: Боковая панель, секция "Питание"
**Что**: Выбран ExeGate ATX-XP350 на 350 Вт. При нагрузке ~130 Вт это OK. Но:
- 350W не хватит для сборки с видеокартой (обычно нужно 500-650W+)
- UI показывает зелёный индикатор без предупреждения о том, что 350W — это абсолютный минимум
**Ожидаемое поведение**: Предупреждение "350 Вт — минимальная мощность. При добавлении видеокарты может потребоваться более мощный БП".
